# 設計文件實作與寫入程式碼的規定

根據公開資訊與業界慣例，設計文件實作與程式碼寫入通常需遵循以下規定：

## 1. 依據設計文件內容實作
- 必須嚴格依據設計文件（如 PRD、SPEC、技術設計書）內容進行程式碼實作。
- 不得擅自更動需求，若有疑問需回報設計負責人。

## 2. 程式碼撰寫規範
- 需遵循團隊或專案既定的程式碼風格（如命名慣例、縮排、註解格式）。
- 需撰寫必要的註解，說明邏輯、參數、回傳值等。
- 重要邏輯需額外補充設計意圖註解。

## 3. 檔案與目錄管理
- 每個功能模組、章節、元件需獨立檔案管理。
- 檔案命名需具備可讀性與一致性。
- 不可將多個無關功能混寫於同一檔案。

## 4. 版本與變更管理
- 每次提交需記錄變更內容（如 Git commit message）。
- 重大變更需附上設計文件修訂說明。

## 5. 測試與驗證
- 程式碼需附帶單元測試或自動化測試腳本。
- 需通過測試後方可進入主分支。

## 6. 文件與程式碼同步
- 設計文件與實作程式碼需保持同步更新。
- 若有差異，需優先修正設計文件或程式碼，並記錄修正原因。

## 7. 安全與合規
- 不得於程式碼中寫入敏感資訊（如密碼、金鑰）。
- 需遵守相關法規與資安規範。
