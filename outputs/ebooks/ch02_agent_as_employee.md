# 第二章：一個 Agent 就是一個員工——你現在是老闆了

## 上一章你知道了

Agent 不是 chatbot。你給目標，它搞定。

但你要怎麼「給目標」？不是亂喊一通。這一章教你：**把 Agent 當員工管。**

---

## 先忘掉 prompt engineering

2023 年流行一個詞叫「prompt engineering」——教你怎麼精心設計一段指令，讓 AI 回得好。

那是在跟 chatbot 對話的時代。現在是 Agent 時代，你要學的不是「怎麼問」，是「怎麼管」。

差別在哪？

| Chatbot 時代 | Agent 時代 |
|-------------|-----------|
| 你設計一句完美的 prompt | 你設計一個工作流程 |
| 你在乎一個回覆的品質 | 你在乎整個任務的完成度 |
| 你的角色：打字員 | 你的角色：老闆 |

---

## 員工思維：你的第一個 Agent

想像你今天請了一個員工。你不會這樣跟他說：

> 「請你幫我……呃……就是那個……用一種溫暖的語氣……大概 200 字……然後要有行動呼籲……」

這是在教 chatbot。對員工，你會說：

> 「我們下個月要推一個新產品。你負責寫商品頁文案。目標客戶是 30 歲上班族。下週五前給我初稿。」

給員工的是**目標、範圍、期限**，不是一句精雕細琢的指令。

Agent 也一樣。

---

## 「給 Agent 的任務」怎麼寫？

三件事就好：

### 1. 目標
你要它做完之後產出什麼？一個檔案？一個網站？一份報告？

✅ 「寫一份 5 頁的市場研究報告，存成檔案」
❌ 「幫我研究一下市場」

### 2. 範圍
它的權限到哪？能做什麼、不能做什麼？

✅ 「用網路搜尋收集公開資料就好，不需要付費報告」
❌ （不解釋範圍，它可能會亂搞）

### 3. 期限
沒期限的東西永遠不會做完。

✅ 「三小時內完成」
❌ 「有空再做」

---

## 真實任務範例

### 任務 1：寫電子書

```
目標：寫一本新手 AI 工具書
範圍：5 章，每章 1500-2000 字，繁體中文，語氣像朋友聊天
    用網路搜尋最新的工具資訊當參考
    不要寫程式碼，不要講技術架構
期限：今天內完成
交付：5 個 .md 檔案，存在 outputs/ebook/ 目錄下
```

### 任務 2：市場研究

```
目標：研究台灣電子書市場的熱門主題 TOP 10
範圍：搜尋 Readmoo、博客來暢銷榜
    整理成表格：書名/主題/價格/評價數
期限：2 小時
交付：一份報告存在 outputs/research/
```

### 任務 3：網站製作

```
目標：做一個個人品牌的單頁網站
範圍：有自我介紹、服務項目、聯絡方式
    簡潔白底黑字風格，不需要花俏動畫
    純 HTML+CSS，不需要後端
期限：3 小時
交付：index.html + style.css，存在 outputs/website/
```

## 「一間公司」不是一個 Agent，是一群

一個 Agent 能做的事有限。但三個 Agent 同時跑：

- Agent A 搜尋資料
- Agent B 寫草稿
- Agent C 校稿排版

你不用等 A 做完才叫 B。**它們同時開工。**

這就是下一章要講的：**多代理系統。**

---

## 這一章的重點

- 把 Agent 當員工管，不是當 chatbot 問
- 給任務三要素：目標、範圍、期限
- 你不是打字員，你是老闆
- 一個人不夠，你需要一個團隊
